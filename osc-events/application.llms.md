# Applications

Code

Some OSC events require an **application process** rather than simple registration. Unlike standard registration, applications involve collecting additional information from applicants and screening submissions to determine eligibility or prioritize participation.

Examples of events that may require applications include:

- Open Science Summer School
- Train-the-Trainer programs

# formr

The OSC uses **formr** to create and manage application forms and surveys.

Formr is a web-based framework for creating surveys, longitudinal studies, and application forms.

You can access the OSC formr instance here: <https://formr.osc.lmu.de/>

To learn more about formr and how it works, consult the official documentation: <https://formr.org/documentation>

> **NOTE:**
>
> At the OSC, formr is primarily used for:
>
> - Event applications
> - Screening applicants
> - Pre-event surveys
> - Post-event feedback surveys

# Creating Application Forms

Application forms in formr are generated from Google Sheets templates found in the [Google Drive](https://drive.google.com/drive/folders/1mGDWgM7ekqP9gZ1u6Q6BlM-wcrZav8gG?usp=drive_link).

The Google Sheets define:

- application questions;
- response options;
- survey flow; and
- other application settings.

When creating a new application form, start by identifying an existing application template that is similar to the event you are organizing and adapt it as necessary.

> **TIP:**
>
> Before creating a new application from scratch, review previous application forms and reuse existing templates whenever possible. This helps ensure consistency across OSC programs and reduces setup time.
